# AI 知识付费直播话术诊断 Agent：MVP 开发上下文

> 本文是项目早期四天开发设想，其中“第一版不做登录、多用户”等内容已经过时。当前真实状态请以 [项目交接与文档写作底稿](project-handoff.md) 和仓库代码为准。

## 当前项目定位

这是一个面向 AI 课程、AI 训练营、AI 陪跑类知识付费直播间的 Web 端话术诊断工具。

用户输入直播链接或上传直播录屏后，系统自动获取直播内容，调用阿里 ASR 转成带时间戳的文字，再结合 AI 知识付费话术框架、风险规则和改写模板，生成可执行的整改建议报告。

第一版只追求垂直闭环，不做平台级稳定性和企业级系统。

## 给客户的大白话说明

这是一个直播话术体检工具。

主播把 AI 课直播链接或录屏放进来，系统会自动把直播里的话转成文字，然后帮主播检查哪些话容易违规、哪些地方转化链路不完整、哪些表达可以改得更安全。最后系统会直接给出能照着用的修改话术。

## MVP 主链路

```text
Web 输入直播链接 / 上传录屏
  -> DouyinLiveRecorder 录制直播片段
  -> ffmpeg 提取音频
  -> 上传到阿里 OSS
  -> 阿里 ASR 转写，拿到文本和时间戳
  -> 用户选择内置框架或上传自定义话术框架
  -> 阿里 text-embedding-v4 向量化检索框架、规则和改写模板
  -> 违禁词扫描 + DeepSeek 语义风险分析
  -> 生成整改建议报告
```

## 当前落地取舍

- 主入口：直播链接录制。
- 兜底入口：上传录屏。
- 直播录制：直接调用 GitHub 开源项目 DouyinLiveRecorder，不做平台稳定性保障。
- ASR：使用阿里 ASR。
- RAG：使用阿里 text-embedding-v4 做向量化，MVP 先用本地向量检索缓存，后续可替换为 Chroma 持久化。
- DeepSeek：接在分析层，结合 ASR 文本、RAG 依据和框架结果判断语义风险与框架缺口。
- Agent：不做复杂自由 Agent，采用固定工作流 + 少量 AI Chain。
- 后端：沿用当前项目已有 NestJS，不另起 FastAPI。
- 前端：沿用当前 React/Vite 页面结构。

## 第一版保留功能

- 输入直播链接并录制固定时长。
- 上传本地直播录屏作为备用入口。
- ASR 转写并展示带时间戳文本。
- 内置 AI 知识付费直播全场转化框架。
- 支持用户上传自定义话术框架。
- 违禁词命中分析。
- 语义风险分析。
- 基于话术框架生成整改建议。
- Web 报告页展示原话、风险、依据和替换话术。

## 第一版不做

- 登录注册。
- 实时直播提醒。
- 视频画面审核。
- 多用户权限。
- 企业后台。
- 多品类模板市场。
- 平台规则自动更新。
- 直播录制稳定性保障。

## 知识库设计

RAG 知识库分三类：

```text
frameworks/
  AI 知识付费直播全场转化框架
  用户上传的话术框架

rules/
  违禁词
  AI 变现承诺风险
  AI 副业案例夸大风险
  AI 工具能力夸大风险
  AI 焦虑式成交风险
  过度逼单风险
  站外导流风险

templates/
  安全改写模板
  弱化承诺模板
  保留促单但降低风险的表达模板
```

默认全场框架：

```text
0-60 分钟：集中输出 AI 干货，让用户觉得有价值
60-84 分钟：课程承接 + 权益说明
84-90 分钟：讲解成功案例继续立人设
90 分钟后：持续成交承接，控制过度逼单和站外交易风险
```

## Agent/Chain 设计

工程上只做固定流程，不做复杂自主规划。

```text
Workflow Orchestrator
  普通后端流程代码，负责录制、上传、ASR、任务状态和调度。

Knowledge Chain
  解析用户上传的话术框架，切块、打标签，用 text-embedding-v4 向量化并写入向量库。

Analysis Chain
  结合 ASR 文本、RAG 检索结果、违禁词命中和 DeepSeek 语义判断，输出风险判断。

Rewrite Chain
  基于风险判断生成更安全、主播能直接照着说的替换话术。
```

## 节奏分析口径

不要只靠字数推断时间。

更准确的口径是：

- 用 ASR 时间戳统计每个话术片段真实耗时。
- 用文本字数判断信息密度。
- 用话术框架判断当前片段属于哪个阶段。
- 用风险词和 CTA 表达判断逼单密度。

最终输出：

```text
阶段耗时
文本密度
匹配话术阶段
缺失内容
风险表达
整改建议
```

## 4 天开发顺序

### Day 1：输入与 ASR

- 调通直播链接录制或本地录屏上传。
- 接入 ffmpeg 提取音频。
- 上传音频到 OSS。
- 调用阿里 ASR，拿到带时间戳文本。

### Day 2：RAG 知识库

- 内置 AI 知识付费直播全场转化框架。
- 建立 text-embedding-v4 向量检索链路，先用本地向量缓存跑通，后续替换 Chroma 持久化。
- 导入违禁词、风险规则和改写模板。
- 跑通一次根据直播片段检索相关知识的流程。

### Day 3：分析与改写

- 实现违禁词扫描。
- 实现语义风险分析 Chain。
- 实现整改话术生成 Chain。
- 后端输出结构化报告 JSON。

### Day 4：Web 闭环和演示

- 做首页输入、转写结果、框架选择、报告页。
- 支持上传自定义话术框架。
- 准备一条可稳定演示的数据链路。
- 补充 README、演示话术和面试讲解材料。
