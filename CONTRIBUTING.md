# Contributing

感谢你关注 AI 知识付费直播复盘项目。

## 开发流程

1. Fork 仓库并创建功能分支。
2. 运行 `npm install` 安装依赖。
3. 从 `.env.example` 创建 `.env.local`，不要提交真实密钥。
4. 完成修改后运行：

```bash
npm run check
npm run build:prod
```

5. 提交 Pull Request，说明改动原因、用户影响和验证方式。

## 贡献范围

优先欢迎以下方向：

- AI 知识付费直播框架、风险规则和可验证案例边界。
- ASR 时间轴、RAG 检索、风险判断和报告追问的测试。
- CSV 或合规第三方数据适配器。
- 无障碍、移动端体验和性能优化。
- 文档、错误提示和部署说明改进。

## 安全与数据

- 不要提交 API Key、AccessKey、Cookie、签名 URL 或账号凭证。
- 不要上传未经授权的直播录屏、逐字稿、用户信息或商业数据。
- 不要把模拟数据标记为真实第三方数据。
- 安全问题请按 [SECURITY.md](SECURITY.md) 私下报告，不要公开利用细节。

## 代码原则

- 保持改动聚焦，优先沿用现有 React、NestJS、LangGraph 和共享类型模式。
- 对外部模型和云服务增加失败降级，不让单个供应商故障拖垮核心流程。
- 新增真实数据源时必须保留来源标签，不得伪造数据真实性。
- 涉及文件路径、上传、URL 和日志时，需要同时补充安全测试。
